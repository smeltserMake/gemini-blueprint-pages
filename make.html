# /make Tool Execution Guide

## URL Construction Pattern
Base URL: https://3vrvux2fh1.execute-api.us-east-1.amazonaws.com/make
Parameters: ?endpoint={endpoint}&method={method}&json={encoded_json}

## Parse User Query
Extract the following from natural language:
- **Action verb**: run, list, get, create, delete, pause, resume
- **Resource type**: scenario, webhook, connection, folder, datastore
- **Resource ID**: numeric or alphanumeric identifier
- **Additional parameters**: any extra data needed

## Complete URL Examples

1. **Run Scenario 123456**
   - Parse: action=run, resource=scenario, id=123456
   - URL: `https://3vrvux2fh1.execute-api.us-east-1.amazonaws.com/make?endpoint=scenarios/123456/run&method=POST`

2. **List All Webhooks**
   - Parse: action=list, resource=webhooks
   - URL: `https://3vrvux2fh1.execute-api.us-east-1.amazonaws.com/make?endpoint=webhooks&method=GET`

3. **Add Record to Users Store**
   - Parse: action=add, resource=record, store=users
   - JSON: `{"name":"John","email":"john@example.com"}`
   - Encoded: `%7B%22name%22%3A%22John%22%2C%22email%22%3A%22john%40example.com%22%7D`
   - URL: `https://3vrvux2fh1.execute-api.us-east-1.amazonaws.com/make?endpoint=datastores/users/records&method=POST&json=%7B%22name%22%3A%22John%22%2C%22email%22%3A%22john%40example.com%22%7D`

## Response Validation
After constructing URL, browse to it and query for:
1. **"Status"** - to confirm execution (✅ or ❌)
2. **"executionId"** - for scenario runs
3. **"error"** - if something failed
4. **"data"** - for GET operations returning data

## Error Handling
If user request doesn't match patterns:
1. Try to infer closest matching operation
2. If ambiguous, use most common interpretation
3. Always execute without asking for clarification
