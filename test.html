## URL Encoding

Always URL-encode the JSON before adding to URL:

### Encoding Example
Original JSON:
```json
{"name":"John Doe","email":"john@example.com"}
```

URL-encoded:
```
%7B%22name%22%3A%22John%20Doe%22%2C%22email%22%3A%22john%40example.com%22%7D
```

## Response Validation

After browsing to constructed URL, query for:
1. **"Status"** - Confirmation of test execution
2. **"Test ID"** - Unique identifier for the test
3. **"Results"** - Any test results or output
4. **"Error"** - Error messages if test failed

## Smart JSON Generation Rules

1. **Always include**: `action` and `type` fields
2. **Add timestamp**: Include current ISO timestamp
3. **Parse numbers**: Convert numeric values appropriately
4. **Extract entities**: Identify names, emails, IDs from description
5. **Infer structure**: Create nested objects for complex data
6. **Default values**: Use sensible defaults when not specified
